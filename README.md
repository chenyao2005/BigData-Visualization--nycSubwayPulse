# 纽约地铁脉搏 (NYC Subway Pulse)

👉 **[点击此处在线访问交互式可视化网页与项目说明文档](https://chenyao2005.github.io/BigData-Visualization--nycSubwayPulse/)**

## 1. 关于本作业
本项目为大数据可视化课程作业二，旨在通过实践，探索并掌握交互式可视化的构建与在线部署（GitHub Pages）。核心目标是设计一个聚焦明确、实现精良的交互式图表，支持用户通过平移、缩放、按需显示细节和动态过滤等交互手段，围绕关键维度进行深度数据探索。

## 2. 本项目解答什么问题？
本可视化项目聚焦于 **纽约地铁通勤潮汐的演变规律**。通过可视化 24 小时内的地铁客流数据，解答以下核心问题：
* **时空分布特征**：早晚高峰期间，曼哈顿商业区与外围住宅区的客流潮汐是如何涌动的？
* **技术普及率差异**：现代移动支付技术（OMNY）与传统磁条卡（MetroCard）的使用比例在不同行政区、不同时间段是否存在显著差异？这反映了怎样的社区通勤特征？

## 3. 如何使用本项目？
* **动态时间探索**：拖动左侧的时间滑块，观察 24 小时内全城地铁站点的客流“潮汐”动画。
* **支付方式对比**：点击左侧的“总客流”、“刷手机”、“刷卡”按钮，一键切换不同的客流指标，观察不同支付方式在空间上的分布差异。
* **区域过滤与空间探索**：使用下拉菜单筛选特定行政区；支持鼠标滚轮缩放与拖拽地图底图进行微观查看。
* **多视图联动分析**：鼠标悬浮（Hover）站点查看具体数值；**点击地图上的任意站点**，左侧辅助视图将自动生成该站点专属的 24 小时客流/支付方式折线趋势图。
